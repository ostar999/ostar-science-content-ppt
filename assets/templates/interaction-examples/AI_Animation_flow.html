1
一句话概括
通过学习海量文本的统计规律，预测下一个最可能出现的词（Token）。这是所有大模型最底层的工作方式。
2
Token 是什么？
文本被拆成词元（Token），每个对应唯一数字ID。如"我喜欢AI"→ [312, 5431, 845]。模型在数字世界里运算。
3
Embedding 词嵌入
把"猫"→ [0.23, -0.54, 1.27, ...]。语义相近的词在向量空间中距离更近，模型因此"理解"词语关系。
4
Attention 注意力
"小明把书放在桌子上，然后他离开了"。Attention 让"他"80% 关注小明，理解上下文指代关系。

训练三阶段

1
预训练 Pre-training
阅读海量互联网内容（书籍·论文·代码·网站），预测下一个 Token → 得到基础模型 Base Model
2
指令微调 SFT
给大量"问题→标准答案"样本，让续写模型学会听懂指令，变成问答助手
3
RLHF 人类反馈强化学习
同一问题 → 多个答案 → 人工排序 → 训练奖励模型 → 强化学习优化 → 更安全、更有帮助
4
推理模型
GPT-5/Claude Opus/DeepSeek-R1：不再直接输出答案，学会分析→规划→验证→输出
!
参数 ≠ 智慧
GPT-2: 1.5亿 → GPT-3: 1750亿 → Llama 70B: 700亿。参数多不一定更聪明，训练数据和训练方法同样重要。
幻觉与局限
"语法正确 + 逻辑合理 + 事实错误"
模型本质是预测最合理的下一个 Token，而不是查询真实数据库。因此会编造不存在的论文、引用、API、人名——这被称为 Hallucination（幻觉），是当前大模型的核心挑战之一。
核心洞察：大模型并不是像人一样"存储知识库然后查询答案"，而是把语言、知识、逻辑和模式压缩进数十亿到数万亿个参数里。当你提问时，它通过 Transformer 的注意力机制分析上下文，并逐个预测最合适的 Token。
